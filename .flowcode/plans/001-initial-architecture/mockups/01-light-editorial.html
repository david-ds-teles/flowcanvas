001-initial-architecture-plan

active plan

Phase 1 — Canvas persistence

Reuse mdview's file API verbatim; the canvas only adds /api/canvas and /api/asset. Path safety stays in guardPath — every route resolves against FLOWCANVAS_ROOT and rejects anything outside it.

server/guard.tstypescript
export function guardPath(root, p) {
  const abs = path.resolve(root, p)   // reuse mdview helper
  if (!abs.startsWith(root)) throw new Error('outside root')
  return abs
}

Phase 2 — Asset pipeline

Inline image nodes resolve through /api/asset against the same root guard; the allowlist covers .png .jpg .svg .webp .avif.
