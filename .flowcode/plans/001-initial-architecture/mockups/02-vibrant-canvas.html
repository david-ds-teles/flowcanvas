Initial architecture — plan

Canvas persistence reuses the existing file API verbatim. The store is a thin slice; routes only add canvas + asset endpoints on top of the proven path guard.

Phase 1 — persistence

Save round-trips through guardPath then the shared POST handler:

typescript 
// canvas persistence reuses the file API
export async function saveCanvas(path: string, doc: FlowcanvasDoc) {
  const safe = guardPath(path)        // reject outside ROOT
  await writeFile(safe, JSON.stringify(doc))
  return { ok: true, revision: doc.revision + 1 }
}

Endpoints added

  • POST /api/canvas — write the board, bump revision
  • POST /api/asset — store inline images under root
