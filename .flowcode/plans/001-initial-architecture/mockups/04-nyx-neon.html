Flowcanvas Design

A node-based canvas that renders, connects, and round-trips flowcode markdown files with an AI agent via an extended-JSONCanvas schema.

Problem Statement

Designing a system out of markdown today is linear and lossy. Files relate to each other — a design references its plan; a module references its tests — but those relationships live as paths buried in frontmatter, never seen as a shape.

Flowcanvas makes the design board itself the medium: a spatial, relational, durable, machine-readable artifact that both a human and an agent read and write through one JSON schema.

Scope

In scope (v0.1):

  • Render flowcode markdown files as canvas nodes showing parsed YAML frontmatter and rendered body.
  • Labeled edges — manually drawn and auto-derived from each file's links: frontmatter.
  • Human comments pinned to a node or canvas coordinates, as flat threads.
  • Export a DesignBrief; import an AgentResponse and merge it onto the board.
The agent loop is the point of the tool — the brief is self-contained, so the agent needs no filesystem access to read it.

Adapter

The adapter maps a FlowcanvasDoc onto React Flow nodes and edges:

// lib/canvas/adapter.ts
export function toReactFlow(doc: FlowcanvasDoc) {
  const nodes = doc.nodes.map((n) => ({
    id: n.id, type: nodeKind(n),
    position: { x: n.x, y: n.y },
    width: n.width, height: n.height,
  }))
  return { nodes, edges: [] } // edges derived from links:
}

Node bodies use a lightweight client renderer; full rehype-shiki highlighting runs only here in the reader.
