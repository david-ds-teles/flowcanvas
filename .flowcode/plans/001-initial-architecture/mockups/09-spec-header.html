Schema & Adapter

The adapter maps a stored FlowcanvasDoc onto React Flow nodes and edges — the single boundary between persisted JSONCanvas and the live board.

Responsibilities

The adapter is pure and deterministic: given the same document it returns the same nodes and edges, so re-renders never drift.

  • Translate each node’s frontmatter into the spec header shown on the card.
  • Derive links: edges, deduplicating against user-drawn ones.
  • Resolve JSONCanvas color presets 1–6 through colorVar(), never inline hex.

Adapter

The mapping reads as plain data in, plain data out:

// lib/canvas/adapter.ts
export function toReactFlow(doc: FlowcanvasDoc) {
  const nodes = doc.nodes.map((n) => ({
    id: n.id, type: nodeKind(n),
    position: { x: n.x, y: n.y },
    width: n.width ?? 360,
    data: { frontmatter: n.fm, status: "active" },
  }))
  return { nodes, edges: deriveLinks(doc) }
}

Node bodies use a lightweight client renderer; full rehype-shiki highlighting runs only here in the reader — proven by the colored tokens above.

The board is the medium: every file is a node, every relationship an edge, every frontmatter a spec header the agent can read without filesystem access.

Edge derivation

Off-board links: targets draw no edge; only in-board references resolve, so the graph never points into nothing.
