# Technical Overview — 003-canvas-foundation Canvas Foundation & Visual Integrity

- Delivers a correct, complete direct-manipulation canvas: every widget (markdown/note/image/link) resizes via a shared `NodeResizeFrame`; a selected edge exposes a visible `EdgeActionBar` (rel ▾ · ✎ Label · ✕); comments are connected to widgets (on-card `CommentBadge`, human-name thread header, inspector list); a visual-integrity pass killed always-on faded rings, recalibrated `--color-secondary` to `#e4c6ff`, role-reduced prose violet, and made the reader frontmatter compact/collapsible.

## Summary

**What:** Four phases delivered a complete direct-manipulation canvas layer on top of existing primitives — universal widget resize (all four node types now carry `NodeResizeFrame`), a discoverable on-edge `EdgeActionBar`, on-card comment badges with click-to-focus inspector navigation, and a cross-cutting visual-integrity pass that removed always-on rings, recalibrated the secondary colour, and made the reader frontmatter collapsible.

**Why:** Flowcanvas v2 shipped the plumbing (typed-relation graph, MCP loop, change-review) but left the canvas itself unusable as a direct-manipulation tool: only group nodes could resize, edge editing was buried in a popover, comments carried no on-widget representation, and always-on faded violet rings made the tool feel unfinished. These gaps had to be solid before the 004 generation-loop redesign landed on top of them.

**Risks:** None structural — this plan is UI-only and operates entirely on existing primitives. `lib/canvas/jsoncanvas.ts`, `mcp/`, and `docs/flowcanvas-agent-contract.md` schema/tools are unchanged; `schemaVersion` stays `0.2`.

**Tech-Debt:** `rgba(221,183,255,*)` chip/pill alpha-fill literals in `app/styles/studio-inspector.css:152` and `app/styles/frontmatter.css:57-58,67-68` remain at the old base value (7-unit delta from recalibrated `#e4c6ff`); accepted per the design's void-wash-literal precedent, perceptually negligible. Pre-existing `eslint-disable-next-line @next/next/no-img-element` at `components/canvas/nodes/image-node.tsx:20` still lacks a ticket reference.

**Next:** (1) Align the `rgba(221,183,255,*)` literals to `rgba(228,198,255,*)` or `color-mix(in srgb, var(--color-secondary) …, transparent)` in a cleanup. (2) Regenerate the three race-affected module docs (`adapter`, `canvas-nodes`, `canvas-shell`) per the concurrent bootstrap's note. (3) Plan 004 (generation loop) builds on this foundation — all four phases are `done` and all gates are green.

## What Was Built

**Phase 1 — Universal Resize** introduced `NodeResizeFrame` (`components/canvas/nodes/node-frame.tsx:20`), a shared wrapper that renders a React Flow `<NodeResizer>` and four source `<Handle>`s as card-siblings (never nested inside the `overflow:hidden` card, as the constraint requires). All four remaining node types — markdown, note, image, link — were updated to accept `selected` and wrap their card in the frame. Markdown stays auto-height (the adapter derives `--fc-body-max` from the stored height; the node itself renders to content height); note/image/link render as fixed boxes matching the dragged dimensions. Resize persists via the existing `setNodeSize` store action. Handles are hidden when `mode === 'comment'` so pin-drop clicks are not swallowed.

**Phase 2 — Edge Action Bar & Explicit Selection** made edge editing discoverable. `LabeledEdge` (`components/canvas/edges/labeled-edge.tsx`) now consumes `selected` from RF `EdgeProps` and portals an `EdgeActionBar` below the label pill when the edge is selected — three controls (rel ▾, ✎ Label, ✕) wired to the existing `setEdgeRel`, `relabelEdge`, and `removeEdgeWriteback` store actions respectively. The delete button was removed from `RelPicker` (single-surface deletion, `data-testid="edge-delete"`). The adapter and `<ReactFlow>` were updated to declare `selectable: true, deletable: true` and `elementsSelectable`/`edgesFocusable` explicitly rather than relying on framework defaults.

**Phase 3 — Comment ↔ Widget Connection** wired comments to their widgets read-side only. A new pure helper `nodeDisplayName` (`lib/canvas/node-name.ts:11`) resolves a node's human name (group→label, link→url, text→first line, file-md→frontmatter.name‖basename, file-other→basename); a new primitive selector `selectNodeCommentCount` (`lib/canvas/store.ts:111`) returns the count of unresolved root node-anchored comments without object-identity churn. The new `CommentBadge` component (`components/canvas/nodes/comment-badge.tsx:14`) renders as a card-sibling inside all four node types, hidden at zero, capped at "9+", with an exact-count `aria-label` and click → `setSelection`. The thread header in `comment-layer.tsx` and the inspector's node-name display in `inspector-rail.tsx` both use `nodeDisplayName` (replacing raw UUIDs). The inspector gained a "Comments on this node" list with click-to-focus (`focusNode`).

**Phase 4 — Visual Integrity & Reader Redesign** was the cross-cutting closer. Always-on faded rings were removed from `.fc-edge-input` (moved to `:focus`), `.fc-note__edit` (rest border changed to `--color-outline-variant`, crisp `--color-primary` focus ring), `.fc-insp__src` (neutral glass surface), and `.fc-agent__ta` (lightened to `--color-surface-low`). `--color-secondary` was recalibrated from `#ddb7ff` to `#e4c6ff` in `app/globals.css`. Prose violet roles were reduced: reader `em` renders in `--color-text-primary`, inline links in `--color-primary`; `h3` retains its violet accent. `FrontmatterView` gained a `collapsible` prop (collapsed default = status pill + first tags + "+N"; `data-testid="reader-fm-toggle"` expander) used by `reader-drawer.tsx`. Dead `.fc-relpick__del` CSS orphaned by Phase 2 was removed from `app/styles/studio-shell.css`.

## Implementation Details

### NodeResizeFrame

**File(s):** `components/canvas/nodes/node-frame.tsx:20`
**What it does:** Shared wrapper rendering `<NodeResizer isVisible={selected && !commentMode} …>` + `{children}` + four source `<Handle>`s as RF fragment siblings. `onResizeEnd` calls `setNodeSize(id, Math.round(d.width), Math.round(d.height))` — writes through the existing store action, no schema change. Handles hidden in comment mode (`s.mode === 'comment'`) so pin-drop clicks reach the canvas.
**Key decisions:** Handles must be card-siblings, not children, because node cards use `overflow:hidden`; the frame fragment mirrors the group-node pattern at `components/canvas/nodes/group-node.tsx:59-66`. Markdown auto-height is preserved by leaving the adapter's `height: undefined` path untouched — only `width` and the `--fc-body-max` clamp are derived from the stored dimension.

```tsx
export interface NodeResizeFrameProps {
  id: string
  selected: boolean
  minWidth: number
  minHeight: number
  children: ReactNode
}
```

### EdgeActionBar (local to LabeledEdge)

**File(s):** `components/canvas/edges/labeled-edge.tsx:103`
**What it does:** Portaled below the label pill when `selected`; three controls wire to `setEdgeRel`/`relabelEdge`/`removeEdgeWriteback`; `data-testid="edge-delete"` is the single delete surface. `RelPicker` container carries `id="fc-rel-picker-{id}"` and both the `rel ▾` button and the label pill carry `aria-controls` pointing at it. Selected stroke thickens from 1.5 to 2.5. Picker closes on deselect via a render-phase `prevSelected` reset (mirrors `comment-layer.tsx`; satisfies `react-hooks/set-state-in-effect`).
**Key decisions:** `EdgeActionBar` dropped the `rel`/`label` props in favour of a local `picker: boolean` for `aria-expanded` — simplification + a11y. Close-on-deselect was implemented as a render-phase reset rather than a `useEffect` because the project lint rule forbids state-in-effect.

### Adapter — edge selectable/deletable

**File(s):** `lib/canvas/adapter.ts:41-42`
**What it does:** `toReactFlow` edge mapping now sets `selectable: true, deletable: true` explicitly (previously relied on RF defaults, making the contract invisible).
**Key decisions:** Explicit flags make the RF contract legible and stable; no behaviour change, but `adapter.test.ts` now asserts these fields.

### canvas-shell — RF selection props

**File(s):** `components/canvas/canvas-shell.tsx`
**What it does:** `<ReactFlow>` receives `elementsSelectable` and `edgesFocusable` as explicit props alongside the existing canvas props.
**Key decisions:** Companion to the adapter change — together they make the full RF selection contract explicit and testable.

### nodeDisplayName

**File(s):** `lib/canvas/node-name.ts:11`
**What it does:** Pure, DOM-free function resolving a canvas node to its human name. Branches: group→`label`, link→`url`, text→`split('\n')[0].slice(0, 50)` (first line, capped), file-md→`meta.frontmatter.name || basename`, file-other→`basename`.
**Key decisions:** Text nodes tightened to strict "first line" semantics during Phase 3 review (Finding 1 fixed); the implementation now matches the design pseudocode and the inline comment.

### selectNodeCommentCount

**File(s):** `lib/canvas/store.ts:111`
**What it does:** Curried primitive selector `(id) => (s) => number` — counts unresolved root node-anchored comments for the given node id. Returns a `number` (not an object) so per-node badge subscriptions do not cause identity churn.
**Key decisions:** Read-only export; no state or action was modified. The selector is used by `CommentBadge` and exercised by `lib/canvas/comment-count.test.ts`.

### CommentBadge

**File(s):** `components/canvas/nodes/comment-badge.tsx:14`
**What it does:** On-card rose corner pill; hidden at 0, `"9+"` above `COUNT_CAP`, exact-count `aria-label`; click stops propagation and calls `setSelection([nodeId])`. Carries `nodrag nopan` RF integration classes and a `💬` icon span.
**Key decisions:** Rendered as a card-sibling inside `NodeResizeFrame` fragments in all four node components (Phase 3 depends on Phase 1 for this structural reason).

### FrontmatterView — collapsible prop

**File(s):** `components/canvas/frontmatter-view.tsx:65`
**What it does:** NEW `collapsible` prop for the `"reader"` variant. Collapsed state (default) shows a dense status-pill + first tags + `"+N"` bar and a `frontmatter ▾` expander (`data-testid="reader-fm-toggle"`, `aria-expanded`). Expanded state reveals the full kv grid + a collapse toggle. "+N" count covers hidden tags + links + restKeys.
**Key decisions:** `collapsible` is reader-only; the `"card"` variant is unaffected. Toggle buttons carry `aria-expanded` (WAI-ARIA preferred over `aria-label` here; deferred a11y note in QA Finding 2).

### Visual Integrity — CSS tokens

**File(s):** `app/globals.css` (`--color-secondary`), `app/styles/edges.css`, `app/styles/nodes.css`, `app/styles/studio-inspector.css`, `app/styles/toolbar.css`, `app/styles/reader.css`, `app/styles/frontmatter.css`, `app/styles/studio-shell.css`
**What it does:** `--color-secondary: #ddb7ff → #e4c6ff` globally. Focus rings on `.fc-edge-input`, `.fc-note__edit`, `.fc-insp__src`, `.fc-agent__ta` moved from base selector to `:focus` pseudo-class. Reader prose roles reduced. Collapsible-frontmatter styles added. Dead `.fc-relpick__del` rules removed from `studio-shell.css`.
**Key decisions:** Token recalibration via `@theme` propagates globally without per-component updates; the seven-unit `rgba()` literal delta was accepted rather than chased as a cleanup item (same category as the pre-existing void-wash literal in `app/globals.css:141`).

## Deviations from Plan

| Plan Spec | Actual Implementation | Reason |
|-----------|-----------------------|--------|
| `EdgeActionBar` props include `rel` and `label` | Props dropped; local `picker: boolean` added for `aria-expanded` | Simplification — the bar reads state from store directly; `picker` needed for a11y expanded state |
| Selected-edge stroke at rest / selected unspecified | Selected stroke thickened 1.5 → 2.5 px | Visual feedback for selection state — logged in Phase 2 deviations |
| Picker close-on-deselect via `useEffect` (reviewer suggestion) | Render-phase `prevSelected` reset | `react-hooks/set-state-in-effect` lint rule forbids state-in-effect; mirrors `comment-layer.tsx` existing pattern |
| `CommentBadge` — no icon specified in plan | `💬` icon span added; `nodrag nopan` RF classes added | Icon aids discoverability; RF integration classes required to prevent badge from interfering with canvas drag/pan |
| `nodeDisplayName` text-node: "first 50 chars, newlines collapsed" (initial impl) | Changed to `split('\n')[0].slice(0, 50)` (strict first-line) | Phase 3 review Finding 1 — honoured design pseudocode and inline comment |
| Collapsible "+N" covers only tags | "+N" also counts hidden links + restKeys | More informative collapsed bar; no spec conflict |
| `app/styles/studio-shell.css` not in Phase 4 plan table | Dead `.fc-relpick__del` rules removed | Phase 2 cleanup follow-through; clean-code, no behaviour change |
