Auth Service

Issues and verifies session tokens for every request entering the platform. The only node permitted to mint a token; everything else calls verify(token) and trusts the boolean.

Responsibilities

  • Mint short-lived JWTs on successful login
  • Verify + refresh tokens on the hot path (p99 < 8ms)
  • Revoke on logout / password change
The API Gateway informs this service on every request; it never writes to the Orders DB directly.

Interface

Exposes POST /token and POST /verify. Session state lives in Redis, not the request path.
