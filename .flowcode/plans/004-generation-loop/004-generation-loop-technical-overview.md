# Technical Overview — 004-generation-loop Markdown-Core Generation Loop

> **Audit:** generated from the code-explorer audit (executor-relayed; the audit agent completed its analysis).

- Delivers the closed Markdown-Core Generation Loop: a markdown design doc → a discoverable Agent Generation Kit → a typed system-design `.canvas`, with the markdown kept as a living, bidirectionally-linked core spine, plus frictionless three-way import (JSON paste / `.canvas` upload / `.canvas` drag-drop). ## Summary

**What:** A markdown design document can now be turned into a fully-typed system-design `.canvas` via any LLM. The Agent Generation Kit (`lib/canvas/generation-kit.ts`) is discoverable over MCP (`get_generation_kit` tool + `flowcanvas://generation-kit` static resource) and via the UI "Generation Kit" tab — it delivers the system prompt, schema contract, worked example, and the author's document in one paste-ready bundle. The resulting board renders each extracted component as a kind-aware widget (`components/canvas/nodes/component-node.tsx`) and the original markdown stays live as a docked `CoreSpine` pane with bidirectional section↔component highlighting. A generated `.canvas` enters the app through `parseFlowcanvasDoc` → `migrateDoc` → `importDoc`, covering paste, upload, and drag-drop.

**Why:** Plan 002 closed the MCP round-trip loop but the agent had no guidance on system-design extraction, the canvas had no semantic node types, and there was no structured path for importing a generated `.canvas`. Plan 004 closes all three gaps additively without breaking any Plan 003 or Plan 002 surface.

**Risks:** `importDoc` always mints a new `<stem>-<rid>.canvas` (no in-place overwrite) — safe but a second import of the same document creates a second file; manual directory cleanup required. Raw `ZodError` messages surface in the import toast (cosmetic; info finding deferred). MCP sidecar (`mcp/flowcanvas-mcp.ts`) is smoke/CDP-only — no direct vitest coverage.

**Tech-Debt:** `COMPONENT_KIND_META.accent` preset ids (`1`–`6`) intentionally do not drive the leaf widget color (only the boundary-group `--node-accent` tint); the leaf color lives in `app/styles/nodes.css` `data-kind` rules. A future accent color change needs a dual edit. The `validate.ts` double-cast (`as unknown as z.ZodType<FlowcanvasDoc>` at `lib/canvas/validate.ts:39`) is a known zod inference gap with `superRefine` + `passthrough` — structurally safe but opaque.

**Next:** Disk-divergence detection banner (Decision 10, deferred from plan 002); visual-parity pixel-diff automation; live MCP-status probe; `clearBoard` spine reset unit test; import toast error cosmetics.

## What Was Built

Plan 004 delivered five cohesive phases over the Plan 003 foundation. Phase 1 laid the additive pure-library base: `ComponentKind` (8-value enum distinct from `NodeKind`, `lib/canvas/jsoncanvas.ts:15`), `SessionMeta.coreDocPath`, the `0.2→0.3` schema ladder (`lib/canvas/migrate.ts`), the single-source kit (`lib/canvas/generation-kit.ts`), slug/outline helpers (`lib/canvas/spine.ts`), and the zod import gatekeeper (`lib/canvas/validate.ts`). `rehype-slug` was inserted after `rehypeSanitize` in the render pipeline so rendered heading `id`s match `slugify()` and `meta.source.anchor` — the three-way slug parity triangle that underpins all bidirectional linking.

Phase 2 routed `meta.kind` on non-group nodes to a new `component-node.tsx` widget (`components/canvas/nodes/component-node.tsx`) — kind glyph, per-kind accent, `§source` chip, 4 handles, resize, comment badge — while keeping `type:'group'` for boundary/colored groups with a `--node-accent` tint (`lib/canvas/adapter.ts:22`). Phase 3 exposed `buildKit` as the 8th MCP tool (`get_generation_kit`) and static resource, and added the "Generation Kit" UI tab in the agent panel. Phase 4 landed the docked `CoreSpine` pane (`components/canvas/core-spine.tsx`) with six new store actions and bidirectional component↔section highlighting driven by `buildSourceIndex`. Phase 5 closed the import path: `importDoc`/`importCanvasFile` in the store, `.canvas`-dispatched `dropzone.tsx`, and paste/upload controls in the import tab.

The delivered user-visible outcome: open a markdown design doc in Flowcanvas, copy the Agent Generation Kit (or invoke `get_generation_kit` over MCP), submit to an LLM, and import the returned `.canvas` — it upgrades to `0.3`, renders as a typed system-design diagram with a live spine, and every component links back to its source section bidirectionally.

## Implementation Details

### Schema — `ComponentKind` + `0.3` Version

**File(s):** `lib/canvas/jsoncanvas.ts:15–79`
**What it does:** Adds `ComponentKind` type union (`lib/canvas/jsoncanvas.ts:15`), `COMPONENT_KINDS` ordered constant (`lib/canvas/jsoncanvas.ts:20`), `ComponentKindMeta` interface and `COMPONENT_KIND_META` record (`lib/canvas/jsoncanvas.ts:26–49`), `NodeMeta.kind?: ComponentKind`, `SessionMeta.coreDocPath?: string`, and widens `SCHEMA_VERSIONS` to `['0.1','0.2','0.3']` (`lib/canvas/jsoncanvas.ts:79`). All additions are optional/additive — no existing board changes behavior at open time until `migrateDoc` runs.
**Key decisions made during implementation:** `ComponentKind` is intentionally distinct from the derived render `NodeKind` (inferred by `nodeKind()` from node type/shape). A comment at `lib/canvas/jsoncanvas.ts:36` documents the dual-consumer split: `COMPONENT_KIND_META.accent` drives only boundary-group `--node-accent` tint via the adapter; the leaf widget color is keyed off `data-kind` CSS rules.

### Generation Kit — Single Source

**File(s):** `lib/canvas/generation-kit.ts:1–50`
**What it does:** `KitSections` interface (`lib/canvas/generation-kit.ts:6`) + `kitSections()` (returns `systemPrompt`, `schemaContract`, `mcpHowTo`, `workedExample`) + `buildKit(markdown?)` (renders the full kit as a paste-ready markdown string, appending the author's document as section 5 when provided). The `schemaContract` subsumes the former `brief.ts` `AGENT_CONTRACT` literal — `AGENT_CONTRACT` in `brief.ts` is now a re-export of `kitSections().schemaContract`.
**Key decisions:** Single-source approach means the MCP tool, the MCP resource, the UI "Copy full kit", `DesignBrief.responseContract`, and `docs/flowcanvas-agent-contract.md` all render from the same `kitSections()` call. The `KIND_CATALOG` constant interpolates `COMPONENT_KINDS.join(', ')` at `lib/canvas/generation-kit.ts:43` so the contract text is always in sync with the type definition.

### Spine — Slug/Outline/Index Helpers

**File(s):** `lib/canvas/spine.ts:1–58`
**What it does:** Pure helpers — `normPath` (path normalization de-duplicated from three call sites, `lib/canvas/spine.ts:9`), `slugify` (fresh `GithubSlugger` per call for determinism, `lib/canvas/spine.ts:15`), `outlineOf` (ATX heading scan with fenced-block toggle, `lib/canvas/spine.ts:21`), `buildSourceIndex` (anchor→nodeIds reverse index, `lib/canvas/spine.ts:36`), `citedDocPaths` (distinct `meta.source.path` values in first-seen order, `lib/canvas/spine.ts:49`).
**Key decisions:** `outlineOf` maintains a fenced-code-block toggle at `lib/canvas/spine.ts:24` so `# Heading` inside a code fence is not extracted as an outline row — discovered during Phase 4 review (Divergence 6). `normPath` was consolidated here from the store, spine, and inspector during Phase 4 review (Divergence 5).

### Validate — Zod Import Gatekeeper

**File(s):** `lib/canvas/validate.ts:1–44`
**What it does:** `flowcanvasDocSchema` (zod discriminated union over the four node types, `schemaVersion z.enum(['0.1','0.2','0.3'])`, edges `.passthrough()`) + `parseFlowcanvasDoc(json): FlowcanvasDoc` (throws `ZodError` on failure). `superRefine` at `lib/canvas/validate.ts:28` rejects `meta.kind:'boundary'` on any non-group node. All four node-union members use `.passthrough()` (`lib/canvas/validate.ts:14–17`) to preserve unmodeled fields on import.
**Key decisions:** `.passthrough()` was added during Phase 1 code review (Review Finding 1 — latent data-loss bug): without it, extension fields like `GroupNode.background`/`backgroundStyle` would be silently stripped on import. The double-cast (`as unknown as z.ZodType<FlowcanvasDoc>` at `lib/canvas/validate.ts:39`) is a known zod inference limitation with `superRefine` + `passthrough`.

### Migrate — Version Ladder

**File(s):** `lib/canvas/migrate.ts:6–19`
**What it does:** `migrateDoc(doc): { doc, migrated }` — `0.1→0.2` bakes `deriveLinkEdges`+`reconcileEdges` into `doc.edges` then bumps `schemaVersion`; `0.2→0.3` is a pure version-bump no-op; `0.3` passes through with `migrated:false`. Shared by `store.load` and `importDoc`.
**Key decisions:** Extracting migration from `store.ts` into a pure module means both the normal open path and the `importDoc` adoption path run identical migration logic. `store.load` persists the bumped version to disk — a one-time on-open upgrade; every board opened after the 004 update becomes `schemaVersion:'0.3'`.

### Adapter — `renderType` Routing

**File(s):** `lib/canvas/adapter.ts:18–22`
**What it does:** `toReactFlow` computes `renderType = n.meta?.kind && n.type !== 'group' ? 'component' : nodeKind(n)` (`lib/canvas/adapter.ts:18`). A non-group node carrying `meta.kind` maps to the `'component'` React Flow node type. A `meta.kind:'boundary'`/colored group keeps `type:'group'` and receives `--node-accent` from `COMPONENT_KIND_META[n.meta.kind].accent` (`lib/canvas/adapter.ts:22`). `autoHeight` is `false` for component nodes (`lib/canvas/adapter.ts:19`) — components keep their authored box.
**Key decisions:** The `'component'` type string is the React Flow key for `ComponentNode` registered in `canvas-shell.tsx`. Keeping `type:'group'` for boundary groups means no new React Flow nodeType is needed for them — the existing `GroupNode` renders the tinted outline via the `--node-accent` CSS var.

### Store — Spine State + Import Actions

**File(s):** `lib/canvas/store.ts:36–41` (spine fields); `importDoc`/`importCanvasFile` (Phase 5)
**What it does:** Five new transient fields: `coreDocBody`, `coreDocDraft`, `coreDocDirty`, `spineHighlightAnchor`, `linkedNodeIds` (`lib/canvas/store.ts:36–41`). Six new spine actions: `setCoreDoc`, `editCoreDoc`, `submitCoreDocEdit` (pendingReview guard → `writeFileApi` → `submitToAgent`), `highlightSpineSection`, `highlightComponents` (calls `buildSourceIndex`), `clearLinkHighlight`. `importDoc(doc, path?)` writes the validated doc to a minted collision-safe `<stem>-<rid>.canvas` then calls `load()` to adopt it (hydrate → `migrateDoc` → persist `0.3` → set active pointer). `importCanvasFile(file)` reads the File, calls `parseFlowcanvasDoc`, then `importDoc`. `load` routes through `migrateDoc`; `newBoard` mints `'0.3'`.
**Key decisions:** `importDoc` write-then-load means the existing `load` path governs hydration and migration order. The collision-safe minted path (`<stem>-<rid>.canvas`) means an import never clobbers an existing board (Divergence 4). `clearBoard` resets all five spine transient fields (Divergence 8).

### Component Node

**File(s):** `components/canvas/nodes/component-node.tsx`
**What it does:** Kind-aware system-design widget — `KindGlyph` (inline SVG per `COMPONENT_KIND_META.glyph`), kind eyebrow (`COMPONENT_KIND_META.label`), one-line role from frontmatter, `§source` chip (`component-source-chip`) that fires `highlightSpineSection(anchor)`, `NodeResizeFrame` (4 handles), `CommentBadge`. `data-testid="component-node"` + `data-kind` + `data-silhouette` attributes drive both testing and CSS accent. Neutral `unknown` fallback for unrecognized kinds.
**Key decisions:** Widget color is driven by `data-kind` CSS rules in `app/styles/nodes.css` (service=cyan, datastore=lime, queue=amber, actor=violet, external=rose, decision=indigo) — not by `COMPONENT_KIND_META.accent`. This is Divergence 1 (documented in `lib/canvas/jsoncanvas.ts:36` and the phase 2 plan deviation note).

### Core Spine

**File(s):** `components/canvas/core-spine.tsx`
**What it does:** Docked flex pane (between canvas and inspector in `canvas-shell.tsx`). Renders the core doc via `/api/render` (rehype-slug heading `id`s). Edit mode: textarea + dirty flag + "Submit changes" (blocked while `pendingReview` — single-open-round invariant). Switcher over `citedDocPaths` → `setCoreDoc`. Section outline from `outlineOf` with per-heading component-count badges from `buildSourceIndex`. Bidirectional: component `§source` chip → `highlightSpineSection(anchor)` scrolls and pulses the section heading; clicking a section outline row → `highlightComponents` pulses linked canvas nodes. Prose relative-links → `focusNode` if the target is on the board.
**Key decisions:** Mounts when any doc is cited (`citedDocPaths.length > 0`), not only when `session.coreDocPath` is set (Divergence 7) — auto-available without an explicit `setCoreDoc`. Flex pane layout means it coexists with the inspector (Divergence 3).

### MCP Sidecar — `get_generation_kit`

**File(s):** `mcp/flowcanvas-mcp.ts`
**What it does:** 8th tool `get_generation_kit({ markdownPath? })` — reads the attached doc via `GET /api/file` when `markdownPath` is provided, returns `buildKit(md)` (or `buildKit()` for the base kit). Static resource `flowcanvas://generation-kit` (`registerResource`, `mimeType:'text/markdown'`) returns `buildKit()` — passive discovery without an active board or a prior `get_board` call.
**Key decisions:** The MCP resource provides a zero-prerequisite discovery path: any MCP-capable harness can dereference the URI to retrieve the full generation kit before interacting with a board.

### CSS

**File(s):** `app/styles/nodes.css`, `app/styles/toolbar.css`, `app/styles/studio-spine.css` (new)
**What it does:** `nodes.css` — `.fc-cmp` glass widget, per-kind accent palette keyed off `data-kind`, cylinder (datastore) + dashed (external) silhouettes, `§source` chip styled as an interactive button, selection ring; `.fc-group` outline honors `--node-accent`. `toolbar.css` — `.fc-tbtn--kit` (lime→cyan) + kit-tab nav styles + `.fc-dropzone__err` import-error toast; dropzone copy mentions `.canvas`. `studio-spine.css` (new, `@import`ed in `globals.css`) — spine pane dimensions + outline + editor + prose + `fc-rf--linked` canvas pulse animation.
**Key decisions:** `studio-spine.css` is a new CSS partial, keeping 004 styles isolated from the five plan-002 studio partials.

## Deviations from Plan

| Plan Spec | Actual Implementation | Reason |
|-----------|-----------------------|--------|
| Widget color driven by `COMPONENT_KIND_META.accent` preset ids | Leaf widget color driven by `data-kind` CSS rules (`app/styles/nodes.css`); `COMPONENT_KIND_META.accent` drives only boundary-group `--node-accent` tint | Nyx preset ids `1`–`6` don't include indigo and don't match the ui-design palette; separate consumers avoid cross-contamination |
| Kit UI as a standalone centered modal | `'kit'` tab in the existing right-drawer agent panel; toolbar button opens it | Keeps the change within the canvas-toolbar module (export-panel + toolbar); acceptable drift per Wave 2 close |
| Spine docked in a single right column replacing the inspector | Spine docks as a flex pane between canvas and inspector; both coexist | ui-design Q: spine + inspector on the right; flex pane requires no `studio-shell.css` grid change |
| `importDoc` adopts the dropped file's exact path | `importDoc` always writes to a minted collision-safe `<stem>-<rid>.canvas` | Never clobbers an existing board on a double-import |
| `normPath` defined in `store.ts` | `normPath` extracted to `lib/canvas/spine.ts:9` and imported by store, spine, and inspector | De-duplicated from three call sites (Phase 4 review finding) |
| `outlineOf` extracts all ATX headings | `outlineOf` skips ATX-heading lines inside fenced code blocks (`lib/canvas/spine.ts:24`) | Prevents `# Heading` inside a code example from appearing as an outline row (Phase 4 review finding) |
| Spine mounts only when `session.coreDocPath` is set | Spine mounts when any doc is cited (`citedDocPaths(nodes).length > 0`) | Auto-available without dirtying the session for boards that cite docs but have not yet set `coreDocPath` |
| `clearBoard` only resets nodes/edges/comments | `clearBoard` also resets the five spine transient fields | Spine state is logically bound to the board; clearing the board must reset the spine |
