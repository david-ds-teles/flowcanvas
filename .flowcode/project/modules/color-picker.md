# ColorPicker

- The shared, reusable colour picker (006-semantic-edges Decision 3): preset chips + a native `<input type=color>` custom swatch + an optional clear. Consumed by the edge style panel and `node-format-bar`.
- Path: `components/ui/color-picker.tsx`; stack: React 19 / TypeScript / client component.
- Public API: `ColorPicker(props: ColorPickerProps)`.
- Status active; generated by human; last updated 2026-06-30.

---

## Purpose

A single colour-selection control reused across the app so colour editing is consistent and "use this colour pick on other components as well" (the operator's spec) is satisfied. Extracted from the former node-format-bar native-picker pattern into a standalone component. Two consumers: `components/canvas/edges/labeled-edge.tsx` `EdgeStylePanel` (per-edge colour override, with a clear that reverts to the flow-type default colour) and `components/canvas/nodes/node-format-bar.tsx` (node text + fill colour).

## Public API

### Functions / Methods

```ts
// ColorPicker(props: ColorPickerProps): JSX.Element — preset chips + native custom swatch + optional clear
interface ColorPickerProps {
  value?: CanvasColor                 // hex "#rrggbb" or a preset id "1".."6"
  presets?: CanvasColor[]             // quick-pick chips (defaults to the 6 nyx presets)
  onChange: (color: CanvasColor) => void
  onClear?: () => void                // shown when clearable (edge: back to the flow-type default colour)
  label?: string                      // aria-label for the group
}
```

## Usage Examples

```tsx
// edge colour override (components/canvas/edges/labeled-edge.tsx EdgeStylePanel)
<ColorPicker value={data.color} onChange={(c) => setEdgeColor(id, c)} onClear={() => setEdgeColor(id, undefined)} label="Edge colour" />

// node text colour (components/canvas/nodes/node-format-bar.tsx)
<ColorPicker value={color} onChange={(c) => setNodeColor(id, c)} onClear={color ? () => setNodeColor(id, undefined) : undefined} label="Text colour" />
```

Demonstrates both consumers; real call sites are `labeled-edge.tsx` (edge colour row) and `node-format-bar.tsx` (text + fill).

## Key Insights

**Conventions & patterns:** chips carry `data-testid="color-chip-<id>"`; the native input is hidden (opacity 0) over a visible custom swatch; `nodrag nopan` so React Flow does not capture drags. Preset vs hex is distinguished by `isHex()` — a preset value seeds the native input via `colorVar(value)`.

**Gotchas & invariants:**
- The native `<input type=color>` only accepts `#rrggbb`; a preset `value` is converted via `colorVar` for the input's `value`, while `onChange` always emits whatever the user picks (a hex).
- `onClear` is rendered only when provided — edges pass it (revert to the type default); nodes pass it conditionally on having a colour set.
