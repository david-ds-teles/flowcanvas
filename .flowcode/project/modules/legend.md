# EdgeLegend

- The on-canvas flow-type legend (006-semantic-edges Decision 2) that DOUBLES AS the edge-type picker: a corner overlay listing the 6 `EdgeType`s (swatch = colour+line+head); when an edge is selected, clicking a row sets that edge's flow type.
- Path: `components/canvas/legend.tsx`; stack: React 19 / TypeScript / client component (inside the React Flow provider).
- Public API: `EdgeLegend()`.
- Status active; generated by human; last updated 2026-06-30.

---

## Purpose

Makes a typed board legible — a persistent key of `EdgeType → {colour, line, head}` so a reader decodes the diagram by appearance. It is also the primary type-setting surface: it reads the currently-selected edge from React Flow's internal store and, when one is selected, each legend row becomes a button that applies that flow type via `store.setEdgeType`. Mounted once in `components/canvas/canvas-shell.tsx`'s centre pane.

## Key Insights

**Conventions & patterns:** the swatch is an inline SVG line (`strokeDasharray` per `line`) + a `<HeadGlyph>` matching the edge end-marker shapes; rows are `<button>`s (`data-testid="legend-<type>"`), `disabled` when no edge is selected, `aria-pressed` for the selected edge's current type. Pinned top-left by `app/styles/studio-edge.css`.

**Gotchas & invariants:**
- The container has `onPointerDown={(e) => e.stopPropagation()}` — WITHOUT it the pointerdown reaches React Flow, deselects the edge before the row's `onClick` fires (disabling the button), and the type never applies.
- Positioned TOP-LEFT, not bottom: bottom-left collides with the RF `<Controls>` and can fall below the fold (the overlay clipped off-screen) — top-left is clear of both Controls (bottom-left) and MiniMap (bottom-right).
- Two `useStore` selectors each scan `s.edges`; both return primitives (no render loop). Could be merged into one selector (a noted micro-optimisation).

---

## Known Gaps

- Legend is read-only for types it cannot set when nothing is selected (rows disabled) — by design.
