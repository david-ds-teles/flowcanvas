## Purpose

`refs.ts` is the single file that owns all document-reference extraction logic for the flowcanvas knowledge graph. Given a source file path, its parsed frontmatter, and its raw Markdown body, it returns a deduplicated array of `DocRef` records — each carrying a kind (`frontmatter` | `link` | `image`), a resolved root-relative target path or verbatim external URL, an optional anchor slug, and an `isExternal` boolean. It deliberately has no filesystem or DOM dependency: callers supply the text, `refs.ts` returns pure data. `brief.ts` is the primary consumer, calling `extractRefs` once per selected node inside `buildBrief` to populate `BriefNode.refs`. `store.ts` imports `DocRef` as a type only.

// Derived from lib/canvas/refs.test.ts:67-79 — "worked example from the plan"
import { extractRefs } from './refs'
import type { DocRef } from './refs'

const refs: DocRef[] = extractRefs(
  'plans/design.md',
  { links: ['plans/plan.md#phase-1'] },
  'See [the plan](./plan.md) and ![arch](../assets/arch.png) and [RF](https://reactflow.dev).',

// refs ===
// [
//   { kind: 'frontmatter', target: 'plans/plan.md',           anchor: 'phase-1',  isExternal: false },
//   { kind: 'link',        target: 'plans/plan.md',           anchor: undefined,  isExternal: false },
//   { kind: 'image',       target: 'assets/arch.png',         anchor: undefined,  isExternal: false },
//   { kind: 'link',        target: 'https://reactflow.dev',   anchor: undefined,  isExternal: true  },

## Key Insights

**Conventions & patterns:**

- **Root-relative vs file-relative split is kind-gated, not syntax-gated.** Frontmatter `links:` entries skip `resolveRel` entirely; their path is stripped of any leading slashes and used as-is (`refs.ts:55`). Body links always go through `resolveRel` (`refs.ts:55`). This is the flowcode file-convention: frontmatter anchors the knowledge graph from the repo root; body links are author-friendly relative paths.
- **Deduplication key is `kind|target|anchor`** (`refs.ts:58`). A frontmatter entry and a body link pointing to the same file with the same anchor are emitted as two separate records (different `kind`). Only exact triple-matches are suppressed.
- **Global regex requires explicit `lastIndex` reset.** `MD_LINK_RE` is declared at module scope with the `g` flag (`refs.ts:14`). `extractRefs` resets `MD_LINK_RE.lastIndex = 0` (`refs.ts:70`) before each call to avoid the classic stateful-global-regex bug where repeated calls on the same module instance skip matches.
- **Pure in-document anchors (`#heading`) are silently dropped** (`refs.ts:73`). Only references that escape the current document (i.e. have a non-empty path component before the `#`) are emitted.
- **External URLs are kept verbatim but anchor-split.** The URL up to `#` becomes `target`; the fragment becomes `anchor` (`refs.ts:47-49`). No normalisation, no trailing-slash stripping.

**Gotchas & invariants:**

- `basePath` must be root-relative without a leading slash. A path like `/plans/design.md` causes `resolveRel` to interpret the leading slash as "already root-relative" (it strips it), which is correct for the frontmatter branch but would silently misbehave for body links if callers passed absolute OS paths. All current callers (`brief.ts:221`) pass the stored `n.file` value which is root-relative by construction.
- Empty string targets (after trimming) are silently skipped (`refs.ts:57`). A frontmatter entry like `links: ['']` or a Markdown link `[text]()` produce no output.
- The `MD_LINK_RE` regex drops `"title"` strings (the optional `"Title"` after the URL inside `()`) — `(?:\s+"[^"]*")?` — so link titles are not captured. Any consumer expecting titles must source them elsewhere.
- `resolveRel` resolves `..` by popping from an array (`refs.ts:23`); it does not guard against over-popping (a path like `../../escape.md` from a root-level file silently produces `escape.md` without error). This is acceptable because inputs come from authored Markdown, not untrusted user input.
- The module was introduced in plan `002` Phase 1 (commit `74f0cc8`) as part of the Decision 9 reference-navigation feature. It has no migration history.

---

## Known Gaps

- No guard against over-popping in `resolveRel` when `..` segments exceed the depth of `basePath` — low risk given trusted inputs, but worth a future hardening note.
- `"title"` strings in Markdown links are silently dropped; no current consumer needs them, but this is undocumented behaviour.
