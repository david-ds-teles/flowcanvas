## Purpose

This module owns the pure diffing logic that powers the post-agent change-review workflow. At Submit time, a `ReviewState` (board snapshot + metadata) is written to a `<board-stem>.review.json` sidecar by `app/api/canvas/review/route.ts`. After the agent round completes, the UI reads back the sidecar, calls `diffDocs(snapshot, currentDoc, roundGeneratedFiles)`, and displays the resulting `ReviewDiff` in the review panel. The module is intentionally free of I/O: it only defines the data shapes and the diff algorithm, so it can be exercised in unit tests without any filesystem or network dependency. Direct consumers are `lib/canvas/store.ts` (the `reviewDiff` store action calls `diffDocs`), `lib/api.ts` (imports `ReviewState` for the client-side HTTP helpers), and `app/api/canvas/review/route.ts` (imports `ReviewState` for the persistence route).

## Usage Examples

```typescript
// Derived from lib/canvas/review.test.ts:44-51
import { diffDocs } from '@/lib/canvas/review'
import type { CanvasNode, FlowcanvasDoc } from '@/lib/canvas/jsoncanvas'

// Snapshot: board at Submit — one node, no edges, no comments
const snapshot: FlowcanvasDoc = {
  nodes: [{ id: 'n-a', type: 'text', text: 'Alpha', x: 0, y: 0, width: 100, height: 50 }],
  edges: [],
  flowcanvas: {
    schemaVersion: '0.2',
    session: { createdAt: '2026-01-01T00:00:00Z', updatedAt: '2026-01-01T00:00:00Z', revision: 1 },
    comments: [],
  },
}

// Current: board after agent round — agent added a second node
const current: FlowcanvasDoc = {
  nodes: [
    { id: 'n-a', type: 'text', text: 'Alpha', x: 0, y: 0, width: 100, height: 50 },
    { id: 'n-b', type: 'text', text: 'Beta',  x: 0, y: 0, width: 100, height: 50 },
  ],
  edges: [],
  flowcanvas: { ...snapshot.flowcanvas },
}

const diff = diffDocs(snapshot, current)
// diff.nodes.added    → ['n-b']
// diff.nodes.updated  → []
// diff.nodes.removed  → []
// diff.edges.added    → []
// diff.comments.added → []
// diff.files          → []  (roundGeneratedFiles defaults to [])
```

## Key Insights

**Conventions & patterns:**

- **Pure-function boundary (Decision 6).** `review.ts` is deliberately I/O-free so the diff logic can be unit-tested without a filesystem or DOM. All persistence (writing/reading/deleting `<board-stem>.review.json`) lives exclusively in `app/api/canvas/review/route.ts`. Never add `import fs` or `import path` to this file. (`review.ts:1` comment; `route.ts:1-8`)
- **Order-insensitive canonical compare.** The private `canon` helper (`review.ts:20-25`) recursively sorts object keys before stringifying. This means a node whose `meta` keys are declared in a different order after serialisation round-trip is not reported as "updated". Any change to `canon` must not break this invariant — the test at `review.test.ts:72-89` explicitly verifies it.
- **Comments are added-only.** `diffDocs` tracks only `comments.added` — there is no `updated` or `removed` bucket for comments (`review.ts:48-49`). The test at `review.test.ts:156-163` asserts this invariant explicitly. Do not add a `removed` bucket without updating every consumer of `ReviewDiff`.
- **`files` is the same reference.** `diffDocs` assigns `files: roundGeneratedFiles` directly — no copy (`review.ts:50`). The test at `review.test.ts:182-189` asserts reference equality. Callers that mutate the returned `diff.files` array will mutate the original.

**Gotchas & invariants:**

- **`pendingReview` / `baseRevision` review-window relationship.** `SessionMeta.pendingReview` (on the board's own `.canvas` JSON, `jsoncanvas.ts:116`) is set to `true` by the MCP tool after an agent round lands. On next board load the UI checks this flag, fetches the `ReviewState` sidecar via `GET /api/canvas/review?path=...`, and calls `diffDocs(reviewState.snapshot, currentDoc, reviewState.roundGeneratedFiles)`. `ReviewState.baseRevision` (`review.ts:5`) and `SessionMeta.baseRevision` (`jsoncanvas.ts:115`) are both set from `session.revision` at Submit time — they serve as an optimistic-concurrency token that the brief uses to detect conflicts (`brief.ts:135`). Conflating these two fields (the sidecar's `baseRevision` vs the live session's `baseRevision`) is a known source of confusion.
- **Persistence path convention.** The sidecar file is named `<board-stem>.review.json` via the transform `board.replace(/\.(canvas|json)$/, '') + '.review.json'` (`route.ts:12`). For a board at `examples/arch.canvas` the sidecar is `examples/arch.review.json`. This path is never known to `review.ts` itself — it is purely a route concern.
- **`ReviewState` is not in `jsoncanvas.ts`.** It lives here in `review.ts` because it is a transport/persistence type that wraps a `FlowcanvasDoc` snapshot. Importing `ReviewState` from `@/lib/canvas/review` (not from `@/lib/canvas/jsoncanvas`) is the correct pattern — `lib/api.ts:2` and `route.ts:5` both follow this.
- **Single commit history.** `git log` for `review.ts` shows one commit (`74f0cc8`), meaning this file was introduced in the v2 System Design Studio integration and has not been modified since. Any future change to `diffDocs` semantics must be coordinated with the store's `reviewDiff` action and the review-panel component.

## Known Gaps

- No `updated` or `removed` tracking for comments — by design (see Key Insights), but worth revisiting if the UI needs to highlight edited or deleted review threads.
- `diffDocs` has no validation: if `snapshot` or `current` is missing `flowcanvas.comments`, it will throw at `review.ts:48`. A null-safety guard may be warranted.
