# Spine

- Pure TypeScript utility module for the Living Core Spine (plan 004 Phase 1) — produces outline data from markdown headings, reverse-indexes canvas nodes back to their source anchors, and enumerates the distinct docs cited by a board.
- Path: `lib/canvas/spine.ts`; stack: TypeScript 5 (pure — no fs, no network, no DOM).
- Public API: `slugify`, `outlineOf`, `buildSourceIndex`, `citedDocPaths`; type `SpineHeading`.

## Purpose

`lib/canvas/spine.ts` is the pure computation layer for the plan 004 Living Core Spine feature. It owns three distinct concerns: (1) deterministic markdown heading → URL-slug conversion (`slugify`, `outlineOf`) that must stay in lockstep with `rehype-slug` rendered heading `id`s and the agent's `meta.source.anchor` convention — this three-way parity is the module's central correctness invariant (Design Q2); (2) building the reverse anchor→nodeId index (`buildSourceIndex`) that drives section↔component bidirectional highlighting in `core-spine.tsx`; and (3) enumerating the distinct source docs cited by a board's nodes (`citedDocPaths`) which feeds the Phase 4 spine-switcher control. No component, route, or store module can compute these correctly without sharing the same `GithubSlugger` logic; this module is the single canonical source.

## Usage Examples

```typescript
// ── Derived from lib/canvas/spine.test.ts:15-27 ──────────────────────────

import { slugify, outlineOf, buildSourceIndex, citedDocPaths } from 'lib/canvas/spine'
import type { CanvasNode } from 'lib/canvas/jsoncanvas'

// 1. Build an outline from a markdown string
const headings = outlineOf('# A\n## Order lifecycle')
// => [
//   { anchor: 'a',               text: 'A',               depth: 1 },
//   { anchor: 'order-lifecycle', text: 'Order lifecycle', depth: 2 },
// ]

// 2. Reverse-index board nodes back to the anchors they were extracted from
const nodes: CanvasNode[] = [
  { id: 'n1', type: 'text', text: '', x: 0, y: 0, width: 1, height: 1,
    meta: { source: { path: 'd.md', anchor: 'order-lifecycle' } } },
  { id: 'n2', type: 'text', text: '', x: 0, y: 0, width: 1, height: 1,
    meta: { source: { path: 'other.md', anchor: 'x' } } },
]
buildSourceIndex(nodes, 'd.md').get('order-lifecycle')  // => ['n1']

// 3. List every cited doc (feeds the spine-switcher Q4 control)
citedDocPaths(nodes)  // => ['d.md', 'other.md']

// 4. One-off slug (three-way parity check):
//    slugify('Order lifecycle') === 'order-lifecycle'
//    === the id= on the <h2> rendered by /api/render (rehype-slug)
//    === the meta.source.anchor the agent stamps
slugify('Order lifecycle')  // => 'order-lifecycle'
```

Demonstrates all four exports against the board fixture used in `lib/canvas/spine.test.ts:15-27`. The slug-parity assertion (test line 7-12) uses `renderMarkdown` to verify the `id=` on the `<h2>` matches `slugify`.

## Key Insights

**Conventions & patterns:**

- *Pure module contract.* Like all `lib/canvas/*` siblings, `spine.ts` has zero side effects — no DOM, no fs, no network. All four exported functions are total (no throws on bad input) and return deterministic values. This is enforced by the project's pure/impure split (`project-overview.md § Code Style`).
- *Fresh slugger for `slugify`, shared slugger for `outlineOf`.* `slugify` constructs `new GithubSlugger()` on every call (`spine.ts:10`) so it is stateless and safe to call in isolation (e.g. from `core-spine.tsx` to resolve a single anchor chip). `outlineOf` uses one `GithubSlugger` instance across the entire document scan (`spine.ts:16`) so repeated headings accumulate the `-1`/`-2` suffixes that `rehype-slug` would produce for the same document — the two sluggers serve different callers and must never be swapped.
- *`buildSourceIndex` is filter-and-group, not a full node scan.* Nodes missing `meta.source`, citing a different `coreDocPath`, or lacking `meta.source.anchor` are silently skipped (`spine.ts:31-33`). A missing anchor key (e.g. `meta: { source: { path: 'd.md' } }` with no `anchor`) is not an error — it is treated as "this node was not extracted from a named section" and excluded from the index.
- *`citedDocPaths` preserves first-seen order.* The function uses a `Set` for dedup but pushes into a plain array (`spine.ts:44-46`), so the returned order reflects board-node order, not alphabetical. `core-spine.tsx`'s spine-switcher renders this list directly (Q4 resolution, design doc line 125).

**Gotchas & invariants:**

- **Three-way slug parity is a hard invariant (Design Q2).** `slugify()` / the agent's `meta.source.anchor` values / `rehype-slug` rendered heading `id`s must all agree. The mandatory test `lib/canvas/spine.test.ts:7-12` asserts this end-to-end. If `render-md.ts` ever moves `rehype-slug` or replaces it with a different slugger, the parity breaks silently at runtime — re-run the test.
- **`outlineOf` does not skip fenced code blocks (known gap, deferred to Phase 4).** A line like ` ```# not-a-heading``` ` inside a fenced code block will be incorrectly parsed as an ATX heading if the regex matches. Fixing this requires tracking fence state in the line scanner. Tracked as a known gap — see Known Gaps below.
- **`github-slugger` is a direct dependency as of plan 004 Phase 1.** Before Phase 1 it was transitive only (via `rehype-slug`). It is now pinned at `^2.0.0` in `package.json`. Do not remove it thinking it is redundant — removing it breaks `spine.ts`.
- **`rehype-slug` must follow `rehypeSanitize` in `render-md.ts`.** The design doc specifies the insertion order explicitly (design doc line 141, step 3). If the pipeline order changes and `rehype-slug` runs before sanitize, the `id` attributes may be stripped by the sanitizer, breaking the parity test.
